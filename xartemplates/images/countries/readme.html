XARAYA TERRITORIAL FLAGS PACK

Version 1.0 [22-Oct-05]
By Pedro Innecco

 

 

ABOUT

These flags has been compiled by Pedro Innecco (that's me!). I am a flag-lover (a hobbyist vexillologist, the one who study flags) and I've been studying flags since I was four years old. I guess the colourful pictures just captivated me as a child.

The flags contained in this package are territorial flags. Meaning that this package contain not only the flags for all countries (up to the release date of this pack), but also for other related regions, such as autonomous states. The rule followed is that if a territory is listed in the ISO3166 list, we will include its respective flag in this pack.

 

USAGE

These flags were designed to be used with the Country List property within Xaraya. By using these flags and the proper templates, a Xaraya-powered site could display the flag of the country/region selected from the Country List property. Here are some examples for its usage:

  • Display a user's flags in the user's profile
  • Display users' flags in the Members List of a site
  1. Unpack these flags under the modules/base/images directory of your Xaraya installation. Alternatively, you could also unpack these flags under the modules/base/images directory of your theme (i.e.: themes/<your-theme-dir>/modules/base/images) however, this would limit the flags usage for the given theme only.
  2. Now you must update the templates showinput-countrylist.xd and showoutput-countrylist.xd for the flags to be displayed. These templates should go under the modules/base/xarproperties directory of your Xaraya installation. Alternatively, you could also place these templates under the modules/base/xartemplates/properties directory of your theme (i.e.: themes/<your-theme-dir>/modules/base/properties) however, you should rename replace the extension of these files from xd to xt. Also, this would limit the user of these templates for the given theme only. Jojodee has created the following templates that one could use as a kick-start:
File showinput-countrylist.xd<!--- License: GPL http://www.gnu.org/copyleft/gpl.html --->
<xar:set name="imagedir">xarTplGetThemeDir().'/modules/base/images'</xar:set>

<xar:set name="countryname">''</xar:set>
<select name="#$name#" id="#$id#" tabindex="#$tabindex#" onChange='document.getElementById("#$id#image").src
= "#$imagedir#" +"/"+
document.getElementById("#$id#").options[document.getElementById("#$id#").selectedIndex].value +"-flag.png";'>

<xar:foreach in="$coptions" value="$coption" >
<xar:if condition="$value eq $coption['id']" >
<option value="#$coption['id']#" selected="selected">#$coption['name']#</option>
<xar:set name="countryname">#$coption['name']#</xar:set>
<xar:else />
<option value="#$coption['id']#">#$coption['name']#</option>
</xar:if>
</xar:foreach>
</select>

<img id="#$id#image" xsrc="#xarTplGetImage($value.'-flag.png','base')#" />
<xar:if condition="!empty($invalid)">
<span class="xar-error">#$invalid#</span>
</xar:if>

There above template is used when the user is editing the country list property (that is, making a selection from the country drop-down list). It will modify the drop-down list containing the country names, so when the user selects a country, the country flag will appear on the right side of the drop-down list.

Note: If you place the flag images in your system directory instead of using override directory in your theme you should adjust the $imagedir variable in line 2 above and set to /modules/base/images/ only.

File showoutput-countrylist.xd<xar:comment> License: GPL http://www.gnu.org/copyleft/gpl.html </xar:comment>
<xar:if condition="!empty($value)" >
<xar:comment>Set the country code to uppercase</xar:comment>
<xar:set name="formattedcode">strtoupper($value)</xar:set>

<xar:comment>For those that want UK instead of GB</xar:comment>
<xar:if condition="$value eq 'gb'">
<xar:set name="formattedcode">'UK'</xar:set>
</xar:if>

<img src="#xarTplGetImage($value.'-flag.png','base')#" alt="#$countryname# flag" /> #$countryname# (#$formattedcode#)
</xar:if>

The above template is used when a user is viewing the country list property (for example, viewing a user's profile that contains the country list property). It will modify the output so the country flag is displayed before the country name and code.

The 11th line of the above code is what formats and display the country name and flag. You can see that the first bit to be displayed is the country flag by using the function xarTplGetImage(). In this function, $value.'-flag.png'  looks for the filename where $value is the ISO3166 code of the given country. The 'base' bit basically tells the xarTplGetImage() function to look for the flag file in within the Base module. Still looking at the flag, you can see that the <img> tag is has an alt attribute that says #$countryname# flag. This would output an alt value such as "Brazil flag", "United Kingdom flag", "Uruguay flag" and so on depending of the country name - so we keep on the XHTML standard which requests all images to have alt tags.

Still on the 11th line of the above code, you can see that after the img tag is closed we have #$countryname# (#$formattedcode#). This would output the country name followed by it's ISO3166 code between parenthesis. For example: Brazil (BR), Uruguay (UY), Israel (IL) and so on.

There are two last things I would like to comment on this template:

  • By following the standards, the country codes should be in lowercase (such as br, uy or il). However, for aesthetics purposes, we would prefer to display them in upper-case. See the 4th line of the above code? That is what converts the country codes to uppercase (therefore BR, UY or IL).
  • On the 6th line of the above code there is a condition <xar:if> - can you see it? What it dos is quite simple: According to the ISO3166, the code for United Kingdom is GB (as in "Great Britain"). However, this doesn't coincide with the UK's Internet's Top Level Domain Code. But the ISO is the ISO, and we wanted to keep with the standard. So what this <xar:if> statement does is to make sure that if the code to be displayed is GB, to output UK instead.

 

POLITICS

If you feel that a given flag is not appropriate, feel free to replace it by following the guidelines below. But please bear in mind that the developers at Xaraya are not interested in engage in politics and they would like to stay as neutral as possible about flags and territorial disputes. Moreover, if you would like to make any POLITE suggestions, please contact Pedro Innecco directly.

 

FLAGS GUIDELINES

  • All flags must be on its original scale, and 16 pixels in height.
  • All flags must be on PNG format.
  • All files must be named as xx-flag.png, where xx is the ISO3166 code for the country or region. For example, ch-flag.png for Switzerland and br-flag.png for Brazil and so on. Additionally, a file called ---flag.png is included so you can use for when there is no Country selected fro a given Country list.
