Counter Property

Description

The Counter dataproperty displays a counter that increments each time a new item is saved. It displays a textbox field that displays the current value of the counter. The value consists of an alphanumeric prefix and an appended number (e.g. "Foo4477").

The property stores its current value in a modvar that it automatically creates when intantiated for the first time. Both the modvar and the initial value for the counter can be configured.

The property is meant to be used as a dataproperty bound to an object and has 3 configuration values, as described below. If the property is used standalone it simply displays the current counter value and does not increment it when saved.

Note: conflicts can arise if this property is used indiscriminately in a multi-user environment.

Usage

Standalone

    <xar:data-input type="counter" />
    <xar:data-output type="counter" />

Bound

    <xar:data-input property="$properties['counter']" />
    <xar:data-output property="$properties['counter']" />

Configuration Values

  • Counter Store: string list [default="dynamicdata,{property ID}"] A comma separated list consisting of a module name and a module variable name. This defines the modvar where the current value of the counter is stored at any time. The default modvar name is the ID of the dataproperty in its object, stored in the dynamicdata module.
  • Counter Value: string list [default=",0"] A comma separated list that consists of a prefix and an initial numeric value of the counter. The numeric value is an integer that can include leading zeros, which will be preserved. The default value corresponds to a numeric counter without prefix, starting at 1.
  • Allow Prefix Change: checkbox [default='false'] This option defines whether the counter prefix can be changed through the interface when adding new items.
